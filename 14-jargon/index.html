    GAME DEVELOPMENT LANGUAGE & CONCEPTS ENCYCLOPEDIA

    Chapter 14 — Industry Jargon, Slang & Acronyms Dictionary

    The Core Mental Shift: What “Jargon” Actually Does

    Industry jargon is often seen as “informal slang.”

    In reality, it is:

    A compression system for complex production meaning into fast, shared signals that reduce communication cost in high-pressure environments.

    A senior studio insight:

    “Jargon exists because full explanations are too slow for real-time production.”

    However, jargon also has a downside:

    • It can hide ambiguity
    • It can mask misunderstanding
    • It can create false confidence in alignment

    Chapter 1: Why Jargon Evolves in Game Studios

    Game development is:

    • Fast-moving
    • Cross-disciplinary
    • Highly iterative
    • Constantly ambiguous

    So language evolves to:

    • Reduce explanation time
    • Encode shared assumptions
    • Signal urgency or risk
    • Compress entire systems into short phrases

    Chapter 2: Core Acronyms (Universal Studio Language)

    KPI (Key Performance Indicator)

    Metrics used to evaluate success.

    Examples:

    • Retention
    • Revenue
    • Engagement

    DAU / MAU

    • DAU = Daily Active Users
    • MAU = Monthly Active Users

    LTV (Lifetime Value)

    Expected total revenue per player.

    ARPU / ARPPU

    • ARPU = Average Revenue Per User
    • ARPPU = Average Revenue Per Paying User

    MVP (Minimum Viable Product)

    Smallest playable version of a system or game.

    POC (Proof of Concept)

    Test to validate feasibility.

    RC (Release Candidate)

    Potential final build.

    GM (Gold Master)

    Final shipped version.

    WIP (Work In Progress)

    Incomplete asset or system.

    ETA (Estimated Time of Arrival)

    When something is expected to be completed.

    TBD / TBA

    • To Be Determined
    • To Be Announced

    Chapter 3: Production Slang (Studio Shortcuts)

    “Scope creep”

    Uncontrolled expansion of features.

    “Feature creep”

    Adding unnecessary features that dilute focus.

    “Ship it”

    Final decision to release.

    “Polish pass”

    Final refinement stage improving quality.

    “Jank”

    Informal term for awkward or unpolished behavior in gameplay or animation.

    Example:

    • Character sliding slightly during idle animation

    “Juice”

    Enhancements that improve game feel:

    • Screen shake
    • Hit flash
    • Sound feedback

    “Spaghetti code”

    Unstructured, tightly coupled code that is hard to maintain.

    “Hard lock”

    Game becomes completely unresponsive.

    “Soft lock”

    Player cannot progress but game still runs.

    “Edge case”

    Rare scenario not normally expected.

    Chapter 4: Engineering Slang

    “Hacky solution”

    Quick fix that bypasses proper architecture.

    “Hotfix”

    Emergency patch deployed quickly.

    “Refactor”

    Rewriting code structure without changing behavior.

    “Tech debt”

    Accumulated cost of shortcuts in code.

    “Brittle system”

    System that breaks easily when modified.

    “Race condition”

    Timing-based bug in concurrent systems.

    “Memory leak”

    Memory not properly released.

    “Null ref”

    Null reference error.

    “Frame spike”

    Sudden performance drop.

    Chapter 5: Design Slang

    “Game feel”

    Overall tactile responsiveness of gameplay.

    “Readability”

    How clearly game state is communicated.

    “Signposting”

    Guiding player attention through design.

    “Skill ceiling”

    Maximum mastery potential.

    “Skill floor”

    Minimum skill required to engage.

    “Onboarding”

    Early learning phase for players.

    “Core loop”

    Primary repeated gameplay cycle.

    “Meta”

    Dominant strategies or systems outside core gameplay loop.

    “Balance pass”

    Adjusting systems for fairness.

    “Nerf / Buff”

    • Nerf = weaken system
    • Buff = strengthen system

    Chapter 6: Art & Animation Slang

    “Placeholder”

    Temporary asset.

    “Greybox”

    Basic level layout without final art.

    “Blockout”

    Early level shape testing phase.

    “Kitbash”

    Reusing asset pieces to build new structures.

    “Silhouette readability”

    Clarity of shape without detail.

    “Tween”

    In-between animation frames.

    “Rig”

    Skeleton structure for animation.

    “IK / FK”

    • IK = Inverse Kinematics
    • FK = Forward Kinematics

    “LOD”

    Level of Detail system.

    Chapter 7: Audio Slang

    “Stinger”

    Short audio cue for events.

    “Mix pass”

    Balancing audio levels.

    “Diegetic”

    Sound that exists in game world.

    “Non-diegetic”

    Sound outside game world (UI, music).

    “Occlusion”

    Sound blocked by environment.

    “RTPC”

    Real-time parameter control.

    Chapter 8: QA Slang

    “Repro”

    Steps to reproduce a bug.

    “Can’t repro”

    Unable to consistently trigger bug.

    “Regression”

    Bug reappears after fix.

    “Blocker”

    Issue preventing progress.

    “Showstopper”

    Critical issue preventing release.

    “Triage”

    Prioritizing bugs.

    Chapter 9: Live Service Slang

    “Whale”

    High-spending player.

    “Dolphin”

    Moderate spender.

    “F2P”

    Free-to-play user.

    “Retention drop”

    Loss of returning players.

    “Content cadence”

    Rate of new content release.

    “Engagement spike”

    Sudden increase in player activity.

    “Meta shift”

    Change in dominant strategies or systems.

    Chapter 10: Studio Communication Shortcuts

    “Greenlight”

    Approval to proceed.

    “At risk”

    Likely delay or failure.

    “Blocked”

    Work cannot continue.

    “WIP build”

    Unfinished playable version.

    “Drop-in”

    Small unplanned addition.

    “Scope hit”

    Feature increased development cost.

    Chapter 11: Hidden Meaning in Common Phrases

    “It’s just a small change”

    Often means:

    Unknown complexity in multiple systems

    “Should be quick”

    Usually indicates:

    Risk is not fully understood

    “It works on my machine”

    Indicates environment mismatch or hidden dependency.

    “We’ll fix it later”

    Often becomes:

    Permanent technical debt

    “Feels off”

    Vague UX or timing issue requiring analysis.

    Chapter 12: Senior Jargon Interpretation Model

    Beginner Thinking

    “What does this term mean?”

    Intermediate Thinking

    “What is this person trying to communicate?”

    Senior Thinking

    “What system complexity, risk, or constraint is being compressed into this shorthand?”

    Expert Thinking

    “What implicit assumptions, dependencies, and organizational pressures are encoded in this single phrase?”

    Key Insight

    In real studios:

    Jargon is not casual language — it is compressed system design communication under time pressure.
