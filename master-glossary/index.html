    GAME DEVELOPMENT LANGUAGE & CONCEPTS ENCYCLOPEDIA

    MASTER GLOSSARY INDEX (Condensed Reference Edition)

    A

    A/B Testing — Comparing two versions of a feature to measure player behavior differences.
    AI (Artificial Intelligence) — Systems controlling non-player behavior (often rule-based, not true intelligence).
    Alpha — Early playable phase where core systems exist but are unfinished.
    Animation State Machine — System controlling animation transitions based on player or AI state.
    ARPU / ARPPU — Revenue metrics per user / paying user.
    Asset Pipeline — Workflow for creating, exporting, and integrating game assets.
    __A_ Pathfinding_ — Algorithm for optimal path navigation in AI systems.
    Affordance — Visual or functional cues indicating interactivity.
    Audio Middleware* — External audio systems (e.g., FMOD, Wwise).

    B

    Bandwidth (Production) — Available working capacity of a developer/team.
    Behavior Tree — Hierarchical AI decision system.
    Beta — Late development stage focusing on stability and bug fixing.
    Bottleneck — System or person limiting production flow.
    Build — Compiled playable version of a game.
    Burndown Chart — Graph tracking remaining work over time.
    Buff — Increase in power or effectiveness of a system.
    Bug — Unintended system behavior.
    Baking (Art) — Precomputing lighting or detail into textures.
    Blocker — Issue preventing progress.
    Branching (Git) — Parallel code development streams.

    C

    Camera Framing — Composition of visual focus in gameplay or cinematics.
    Cognitive Load — Mental effort required to process information.
    Collision Detection — System determining object intersections.
    Core Loop — Repeated gameplay cycle driving engagement.
    Critical Path — Sequence of tasks determining project timeline.
    Crunch — Excessive overtime during development (controversial).
    Culling (Occlusion) — Not rendering unseen objects.
    Client-Server Model — Multiplayer architecture with authoritative server.
    Cooldown — Time delay before ability reuse.
    Crafting System — Resource conversion mechanics for progression.
    Creative Direction — Unified vision guiding all disciplines.

    D

    DAU / MAU — Daily / Monthly Active Users.
    Data-Oriented Design (DOD) — Performance-focused programming model.
    Debugging — Process of identifying and fixing errors.
    Delta Time — Time between frames for consistent simulation.
    Design Pillars — Core principles guiding game design decisions.
    Dependency — Task or system required before another can function.
    Diegetic UI — UI existing within game world.
    Difficulty Curve — Progression of challenge over time.
    Draw Call — GPU instruction to render objects.
    DLC — Downloadable Content.
    Dissonance (Ludonarrative) — Conflict between gameplay and story.

    E

    Edge Case — Rare or extreme system condition.
    ECS (Entity Component System) — Data-driven architecture pattern.
    Emergent Gameplay — Unexpected outcomes from system interactions.
    Engagement Loop — Systems designed to retain players.
    Engine — Core software framework for game creation.
    Event System — Decoupled system communication model.
    Exploit — Unintended player advantage in system.
    Early Access — Public pre-release version for feedback/testing.

    F

    F2P (Free-to-Play) — Monetization model without upfront cost.
    Fidelity — Level of detail or realism.
    Frame Rate (FPS) — Number of rendered frames per second.
    Frame Time — Time taken to process a single frame.
    Friction (UX) — Barriers slowing user interaction.
    Foley — Recorded real-world sound effects.
    FOV (Field of View) — Camera viewing angle.
    Feedback Loop — System where output influences future input.
    Feature Creep — Uncontrolled feature expansion.
    Flow State — Optimal player engagement state.
    Funnel (Conversion) — Player progression through stages.

    G

    Game Loop — Continuous update cycle of game simulation.
    Gameplay Feel (“Game Feel”) — Responsiveness and tactile quality of interaction.
    Gacha — Randomized reward monetization system.
    GAAS (Games as a Service) — Continuous live-updated games.
    Gestalt Principles — Visual perception rules in design.
    GPU Bound — Performance limited by graphics processing.
    Gold Master (GM) — Final release version.
    Greenlight — Approval to begin full production.

    H

    Haptics — Vibration-based feedback.
    Headless System — System without visual representation (backend logic).
    Hitbox — Area used for detecting collisions in gameplay.
    HUD (Heads-Up Display) — On-screen gameplay information.
    Hotfix — Urgent patch for critical issues.
    HRTF — Audio system simulating human spatial hearing.

    I

    IK / FK — Inverse / Forward Kinematics in animation.
    Input Buffering — Allowing delayed input execution.
    Instance (Game Object) — Runtime object created from a template.
    Iteration — Repeated refinement cycle in development.
    Inventory System — Item management structure.
    IP (Intellectual Property) — Owned franchise or universe.

    J

    Jank — Unpolished or awkward system behavior.
    Juice — Enhanced feedback (shake, sound, effects).
    Jira Ticket — Task tracking entry in production systems.

    K

    KPI (Key Performance Indicator) — Success measurement metric.
    Kitbash — Reusing modular assets creatively.

    L

    Latency — Delay between input and response.
    LOD (Level of Detail) — Reduced complexity models at distance.
    LiveOps — Ongoing live game management.
    LTV (Lifetime Value) — Total revenue per player over time.
    Ludonarrative — Relationship between gameplay and story.
    Lightmap — Precomputed lighting texture.

    M

    MAU — Monthly Active Users.
    Meta (Game Meta) — Dominant strategies in gameplay.
    Middleware — External tools integrated into engine.
    Memory Leak — Unreleased memory causing performance issues.
    MVP (Minimum Viable Product) — Minimal playable system.
    Matchmaking — Player pairing system in multiplayer.
    Microtransaction — Small in-game purchase.
    Mise-en-scène — Visual arrangement in a scene.

    N

    NavMesh — Navigation system for AI movement.
    Netcode — Networking implementation for multiplayer.
    Nerf — Reduction in system power.
    Node System — Graph-based logic structure.
    Non-Diegetic UI — UI outside game world.

    O

    Occlusion Culling — Not rendering hidden objects.
    OOP (Object-Oriented Programming) — Class-based programming model.
    Onboarding — Early player learning experience.
    Open World — Non-linear exploration structure.
    Overhead — Extra system cost beyond core function.

    P

    Patch — Game update.
    Pacing — Control of gameplay rhythm.
    PBR (Physically Based Rendering) — Realistic material system.
    Pipeline — Workflow system for production.
    Playtest — Player testing session.
    Player Agency — Degree of control player has.
    POC (Proof of Concept) — Feasibility prototype.
    Postmortem — Analysis after project completion.
    Procedural Generation — System-generated content.
    Production — Phase of active game development.
    Profiling — Performance analysis process.
    P2W (Pay-to-Win) — Monetization giving gameplay advantage.

    Q

    QA (Quality Assurance) — Testing and validation process.
    QC (Quality Control) — Output verification testing.
    Quest System — Structured objective system.

    R

    Raycasting — Line-based collision detection.
    Repro Steps — Steps to reproduce a bug.
    Retention — Player return rate.
    Regression — Reappearing bug after fix.
    Rigidbody — Physics-simulated object.
    RTPC — Real-time audio parameter control.

    S

    Scope Creep — Uncontrolled feature expansion.
    Shader — GPU visual effect program.
    Silhouette Readability — Recognizability of shape.
    Skill Ceiling/Floor — Mastery range of a system.
    Spawn System — Entity generation logic.
    Stinger (Audio) — Short audio cue.
    State Machine — System with defined behavior states.
    Stickiness — Engagement metric.
    Sprint — Timeboxed development cycle.
    Steam/Platform Holder — Distribution platform provider.
    Sync (Meeting) — Cross-team alignment session.

    T

    Tech Debt — Accumulated inefficiencies in code/design.
    Tick Rate — Frequency of simulation updates.
    Telemetry — Player behavior data collection.
    TDD (Technical Design Document) — Engineering system spec.
    Triage — Bug prioritization process.
    Texture Mapping — Applying images to 3D surfaces.

    U

    UI (User Interface) — Visual interaction layer.
    UX (User Experience) — Overall interaction experience.
    Utility AI — AI decision system based on scoring.

    V

    VFX (Visual Effects) — Particle and visual simulation effects.
    Vertical Slice — Fully polished representative gameplay segment.
    Velocity (Scrum) — Work output per sprint.
    Voice Limiting — Restricting simultaneous audio sources.

    W

    Waypoint — Navigation marker in level design.
    WIP (Work In Progress) — Incomplete asset/system.
    Wwise / FMOD — Audio middleware tools.

    X / Y / Z (Advanced / Rare Terms)

    Z-Depth — Distance from camera in rendering.
    Zone Design — Structuring game world regions.
    Z-fighting — Rendering artifact from overlapping surfaces.

    FINAL NOTE (Professional Interpretation Layer)

    This glossary is not just vocabulary.

    In professional game development:

    Each term is a compressed representation of systems, workflows, and constraints that only fully make sense inside a production environment.
